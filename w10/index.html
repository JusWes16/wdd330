    Client-side validation

    Form validation is anything that requires a certain form field be filled or that it be filled with the correct information or the information in the format needed by the site.

    Reasons to use form validation: get the correct data from the user in the correct format, to protect the users data, to protect ourseleves (the programmers or the owners of the site) from malitious data.

    There are 2 types of client side validation, HTML5 built in validation and writing your own validation using Javascript code.

    Using fetch

    You can use fetch() to POST JSON data by setting a 'Content-Type' header to 'application/json'.

    You can also use fetch() to upload files by using an input tag with the type 'file' and FormData().

    You can create your own headers using the Headers() constructor. The contents of a headers can be queried and retrived using the myHeaders.get method.

    You can use Polyfill to use fetch in any browser that does not support the use of fetch.
