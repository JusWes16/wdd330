Chapter 10

Typical errors:

  • System error – there is a problem with the system or external devices
  • Programmer error – the program has some sort of error because of how it was written. (syntax, typo, etc.)
  • User error – incorrect data from the user that the program didn’t expect and could handle
Different types of errors that can be returned:
  1. Exceptions: returns a return value to help deal with the error
  2. Stack Traces: a sequence of functions or methods that lead to where the error occurred
  3. Warnings: a potential error in the code that isn’t a big enough deal to crash the program
Strict Mode throws more errors and warning because it is considering old practices and methods as and error or warning.
Debugging in the browser:
  1. Using alerts
  2. Using the console
  3. Breakpoints in the console
  4. Error objects
You can also use exception handling with try, catch and finally blocks
